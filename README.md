# MDM-TA2-CASE-STUDY-HARSHIT-SURYAWANSHI-142
This project is a case study of an Order Tracking System inspired by Amazon's logistics and delivery workflow.
It explains how customers can track their orders in real-time, how order data moves through different stages, and how the system ensures transparency between sellers, warehouses, delivery partners, and customers.
The case study focuses on system design, workflow, database structure, and tracking logic rather than Amazon’s proprietary implementation.
🎯 Objectives
To understand how large-scale e-commerce platforms manage order tracking
To design a simplified and efficient tracking workflow
To study status updates from order placement to final delivery
To analyze database and system components required for tracking systems
🛒 Order Tracking Workflow
The system follows a structured order lifecycle:
Order Placed
Customer places an order successfully.
Order Confirmed
Payment verified and order confirmed by seller.
Order Packed
Item packed and ready for shipment.
Shipped
Courier partner picks up the package.
Out for Delivery
Package is out for final delivery.
Delivered
Order delivered successfully to customer.
Return / Replacement (Optional)
Customer may request return or replacement.
